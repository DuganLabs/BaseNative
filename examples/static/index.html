BaseNative

Angular-inspired control flow over native <template> elements. Semantic HTML as truth. Modern CSS as design. A sprinkle of JS as glue.

~120 lines of runtime · zero dependencies · no build step

Philosophy

HTML is the component

Semantic elements are your API surface. No wrapper elements. No synthetic component tags. <article>, <section>, <nav> — they already mean something.

CSS is the design system

Cascade layers, container queries, :has(), logical properties. The platform solved layout and state-driven styling. Let it.

Templates are the primitive

The <template> element is the browser's own inert fragment. Clone it, bind it, flow-control it — no virtual DOM needed.

Control flow is the DX

@if, @for, @switch — Angular proved this syntax reads like markup, not code. We bring it to native templates.

Architecture

Live Demos

@if — Conditional Rendering

<template @if="visible()">
  <p>{{ message() }}</p>
</template>
<template @else>
  <p>(nothing to see)</p>
</template>

@for — List Rendering

<template @for="item of items(); track item.id">
  <li>{{ item.name }}</li>
</template>
<template @empty>
  <li>No items yet</li>
</template>

@switch — State Matching

<template @switch="status()">
  <template @case="'active'"> ... </template>
  <template @case="'idle'">   ... </template>
  <template @case="'error'">  ... </template>
</template>
